Siendo $test_n$ el conjunto de test de la ejecucin n (analogamente para $train_n$). Lo que tenemos para la estrategia simple es que la interseccin entre $test_i$ y $test_j$ no es necesariamente vaca. Mientras que para la estrategia cruzada se garantiza que sea vaca. Adems, en la estrategia cruzada, se tiene que la unin de todos los $test_i$ sea igual al conjunto total.

Entonces, una ventaja de la validacin cruzada es que permite evaluar la precisin del modelo con todos los datos del dataset, mientras que, por la aleatoriedad de la eleccin, en la validacin simple puede haber datos que en ninguna iteracin estuviesen en el conjunto de test.

Por otra parte, la estrategia simple es ms sencilla de implementar. Tambin, con nEjecuciones = 1 es ms rapida que la validacin cruzada.

En caso de tener suficientes datos no es necesario utilizar validacin cruzada ni nEjecuciones > 1 ya que el conjunto de test tendr datos suficientes para dar una estimacin realista de la precisin del modelo. Por otro lado, la validacin cruzada es muy til en un conjunto de datos pequeo ya que la desviacin tpica (decreciente con el tamao de la muestra) podra ser relativamente grande y, por tanto, sera necesario promediar los errores para dar una estimacin del error ms realista.

def get_ROC_data(conf_data, clases, ocurrencias_clases):
    
    matriz_conf = np.zeros((2, 2))
    matriz_conf[0][0] = ocurrencias_clases[clases[1]]
    matriz_conf[0][1] = ocurrencias_clases[clases[0]]
    tpr = np.empty(len(datostest))
    fpr = np.empty(len(datostest))

    for i in range(len(conf_data)):
        if conf_data[i][1] == clases[0]: # Dato negativo
            matriz_conf[1][1] += 1 #TN
            matriz_conf[0][1] -= 1 #FP

        else: # Dato positivo
            matriz_conf[1][0] += 1 #FN
            matriz_conf[0][0] -= 1 #TP


        tpr[i] = matriz_conf[0][0]/(matriz_conf[0][0]+matriz_conf[1][0])

        fpr[i] = matriz_conf[0][1]/(matriz_conf[0][1]+matriz_conf[1][1])
    
    return tpr, fpr

def get_matriz_conf(datostest, prediccion, clases):
    matriz_conf = np.zeros((2, 2))
    
    for i in range(len(datostest)):
        if prediccion[i] == clases[0]:
            if datostest[i][-1] == clases[0]: # TN
                matriz_conf[1][1] += 1
            else: # FN 
                matriz_conf[1][0] += 1
        else:
            if datostest[i][-1] == clases[0]: # FP
                matriz_conf[0][1] += 1
            else: # TP
                matriz_conf[0][0] += 1
    
    return matriz_conf

def AUC(fpr, tpr):

    auc = 0
    for i in range(len(tpr)-1):
        auc += (fpr[i]-fpr[i+1])*tpr[i]
    
    auc += fpr[-1]*tpr[-1]
        
    return auc

Comentarios respecto a modificaciones en el diseo de clases

En la clase EstrategiaParticionado, en la funcin creaParticiones cambiamos los parametros recomendados por un parametro n_datos, ya que en nuestra opinin es el nico parmetro necesario para crear los conjuntos de indices. Con esto, deja de ser necesario pasar a la funcin una instancia de la clase Datos.

A la hora de entrenamiento y clasificacin, modificamos el conjunto de datos a un formado adecuado (cambiar los atributos nominales a un entero siguiendo el orden alfabtico) utilizando el diccionario fuera de las funciones de la clase Clasificador. Esto nos parece ms adecuado en cuanto a la separacin las funcionalidades.

Tambin, tenemos el criterio de transformar la clase a su valor en el diccionario, sea el atributo nominal o numrico ya que la clase siempre es discreta. Esto permite simplificar el codigo gracias a la posible indexacin array[valor1_clase].

Asumimos la equivalencia entre nominalAtributos de la clase Datos y el parmetro atributosDiscretos que se pasa a las funciones de la clase Clasificador (con esta suposicin resulta coherente transformar los valores de clase, atributo discreto, siempre a un entero entre 0 y n_valores).

En la funcin clasifica aadimos el atributo alpha, el parmetro de la correccin de Laplace.

Por ltimo, cambiamos los atributos que se pasan a la funcin validacion de la clase Clasificador aadiendo todos aquellos atributos necesarios para llamar a las funciones entrenamiento y clasifica. Tambin, eliminamos el parametro clasificador llamando a la funcin validacin como instancia_clasificador.validacion(...).
